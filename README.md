## چکیده

این پژوهش با هدف **ارتقای خودکار یک مخزن پایهٔ NLP** و تبدیل آن به یک ابزار حرفه‌ای، بدون وابستگی به کتابخانه‌های خارجی و تنها با استفاده از زبان پایتون و JavaScript خالص انجام شد.  
محیط توسعه یک دستگاه تلفن همراه مجهز به **Termux** با اینترنت ناپایدار و محدود بود؛ با این وجود، تمامی مراحل شامل تحلیل نحوی، بهینه‌سازی کد، افزودن الگوریتم‌های جدید و ایجاد رابط کاربری تحت وب با موفقیت به انجام رسید.

الگوریتم‌های نهایی شامل **تحلیل احساسات مبتنی بر لغت‌نامه (فارسی و انگلیسی)**، **خلاصه‌سازی استخراجی با وزن‌دهی TF-Position**، **استخراج کلمات کلیدی با امتیاز ترکیبی** و **تصحیح فاصله‌گذاری و نیم‌فاصلهٔ فارسی** می‌باشند.  
پروژه با یک مجموعه تست جامع اعتبارسنجی شده و نسخهٔ وب آن از طریق GitHub Pages در دسترس عموم قرار دارد.

**کلیدواژه‌ها:** پردازش زبان طبیعی، متن‌کاوی فارسی، Termux، بهینه‌سازی کد، توسعهٔ خودکار، GitHub Pages

## ۱. مقدمه

پردازش زبان طبیعی (NLP) یکی از شاخه‌های اصلی هوش مصنوعی است که امکان تعامل رایانه با زبان انسان را فراهم می‌کند.  
با توجه به محدودیت ابزارهای بومی برای زبان فارسی و نیاز به راه‌حل‌های سبک و مستقل، این پروژه با دریافت یک مخزن خام از GitHub و اعمال چندین مرحله توسعهٔ خودکار، به یک محصول کامل تبدیل شد.

مخزن اولیه از آدرس زیر clone گردید:

```

https://github.com/tetrashop/nlp-project.git

```

هدف اصلی، **ایجاد یک سیستم NLP مقاوم** بود که روی حداقل منابع (Termux بدون دسترسی به pip/npm) قابل اجرا باشد و در عین حال از استانداردهای مهندسی نرم‌افزار پیروی کند.

## ۲. روش‌شناسی

فرآیند توسعه در شش فاز اصلی انجام شد:

### ۲.۱. جمع‌آوری و تحلیل اولیه کدها
- تمام فایل‌های برنامه‌نویسی مخزن (۲۲۵۷ فایل) در یک فایل متنی تجمیع شدند.
- یک تحلیل‌گر نحوی (syntax checker) با استفاده از ماژول `ast` پایتون و `bash -n` برای فایل‌های شل توسعه داده شد (به دلیل عدم امکان نصب linterهای استاندارد).

### ۲.۲. بهینه‌سازی خودکار کد
یک Code Optimizer مبتنی بر `ast` پیاده‌سازی گردید که موارد زیر را روی فایل‌های پایتون اعمال کرد:
- حذف `else`های زائد پس از `return`، `raise`، `continue` و `break`.
- تبدیل حلقه‌های `for` با `append` به list comprehension.
- جایگزینی `str.format()` با f-string در الگوهای ساده.

### ۲.۳. افزودن ماژول‌های جدید
اسکریپت `enhance_repo.py` ماژول‌های زیر را به پروژه تزریق کرد:
- **`logger.py`**: سیستم گزارش‌گیری استاندارد با قابلیت خروجی فایل.
- **`config.py`**: مدیریت پیکربندی از فایل `config.json`.
- **`nlp_enhancements.py`**: هستهٔ NLP با توابع:
  - `analyze_sentiment`: تحلیل احساسات با لغت‌نامه‌های گستردهٔ فارسی و انگلیسی.
  - `extractive_summary`: خلاصه‌سازی استخراجی با امتیازدهی TF-Position.
  - `extract_keywords`: استخراج کلمات کلیدی با امتیاز ترکیبی فراوانی-طول.
  - `fix_persian_spacing`: تصحیح فاصله‌های مجازی و نیم‌فاصله‌ها.

همچنین تمام فایل‌های `.py` موجود با افزودن docstring، تبدیل `print` به `logger.info`، انتقال کد سراسری به تابع `main()` و قرارگیری در بلوک `try...except` ارتقا یافتند.

### ۲.۴. طراحی رابط کاربری وب
یک رابط گرافیکی کاملاً سمت کلاینت با HTML, CSS و JavaScript ایجاد شد که هر چهار الگوریتم NLP را در مرورگر اجرا می‌کند.  
این واسط بدون نیاز به سرور روی GitHub Pages مستقر گردید.

### ۲.۵. تست و تضمین کیفیت
مجموعه‌ای از تست‌های خودکار (`test_comprehensive.py`) برای پوشش موارد مثبت، منفی و edge-caseها نوشته شد.  
تمامی تست‌ها با موفقیت پاس شده و پروژه به عنوان **بدون باگ** تأیید گردید.

### ۲.۶. ادغام و استقرار
کدهای توسعه‌یافته طی یک شاخهٔ `enhanced` ایجاد، تست و سپس با شاخهٔ `main` ادغام شدند.  
نسخهٔ وب روی GitHub Pages و مخزن اصلی روی GitHub منتشر گردید.

## ۳. خطاهای رخ‌داده و راه‌حل‌ها

در طول توسعه، چالش‌های متعددی پیش آمد که به شرح زیر مدیریت شدند:

| خطا | علت | راه‌حل |
|-----|------|--------|
| `ImportError: cannot import name 'logger'` | تزریق اشتباه `from logger import logger` در خود ماژول `logger.py` | افزودن شرط عدم تزریق در `logger.py` و بازنویسی دستی ماژول |
| `FileNotFoundError: './nlp-project'` | مسیر نادرست `SRC_DIR` در اسکریپت توسعه | تغییر به `SRC_DIR="."` |
| `cp: cannot stat` برای پوشهٔ enhanced | استفاده از مسیرهای نسبی اشتباه | استفاده از `~/nlp_project_enhanced` و انتقال دستی |
| نبود linterهای استاندارد (pylint, eslint) | تحریم اینترنت و عدم امکان نصب بسته‌های pip/npm | پیاده‌سازی تحلیل‌گر نحوی با `ast` و `bash -n` |
| `bash: command not found` در Termux | مسیرهای غیراستاندارد bash در اندروید | استفاده از `which bash` و اولویت‌دهی به `$PATH` |
| شکست regex در شناسایی کلمات دارای نیم‌فاصله | `\w` از نیم‌فاصله (U+200C) پشتیبانی نمی‌کند | اصلاح regex به `[\w\u200C\u200D]+` در پایتون و JavaScript |
| عدم برقراری ارتباط با سرورهای Google/Vercel | محدودیت شبکه و احراز هویت دومرحله‌ای | استقرار استاتیک روی GitHub Pages به جای Vercel |
